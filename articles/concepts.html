    How does it work?

    Before I answer how it works, I would like to answer how it's more performant than the built-in StringBuilder?

    The basic idea is to use a ref struct. This ensures the ValueStringBuilder will live on the stack instead of the heap. This allows the use of stack-allocated Span<T> using stackalloc. Otherwise, it makes use of ArrayPool to reuse arrays. This allows ValueStringBuilder to gain performance with low allocations. With this approach, some limitations arise. Head over to the Known Limitations to know more.
